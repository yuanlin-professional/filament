# BufferPoolAllocator.h

## 文件概述

实现一个简单的缓冲区池分配器模板。池有固定容量，池中所有缓冲区具有相同大小（由最近一次最大分配请求决定）。缓冲区大小只会增长不会缩小，除非显式重置。适用于频繁分配/释放相同大小缓冲区的场景。

## 核心类/结构体/函数

### `BufferPoolAllocator<POOL_SIZE, ALIGNMENT, AllocatorPolicy, LockingPolicy>` 模板类

- **`get(size_type size)`** -- 从池中获取一个至少 `size` 字节的缓冲区。如果请求大小超过当前池缓冲区大小，清空池并以新大小重建。如果池为空则新分配。
- **`put(void* buffer)`** -- 将缓冲区归还到池中。如果缓冲区小于当前池大小或池已满，则直接释放。
- **`reset()`** -- 清空池并将大小重置为 0。

### 内部结构

- **`Header`** -- 分配在每个缓冲区头部的元数据结构，记录缓冲区的实际大小。

## 关键实现逻辑

- **大小对齐**：分配大小向上对齐到 4096 字节（`ALLOCATION_ROUNDING`），减少 `malloc` 调用次数。
- **池增长策略**：当请求大小超过当前池缓冲区大小时，清空所有旧缓冲区并以新的（更大）大小创建，实现"只增不减"的池大小策略。
- **头部元数据**：每个缓冲区前置一个 `Header` 结构记录大小，归还时通过指针回退来读取头部信息。
- **线程安全**：所有操作使用 `LockingPolicy` 模板参数进行同步控制。

## 依赖关系

- `utils/FixedCapacityVector.h` -- 固定容量的向量容器
- `utils/compiler.h` -- 编译器工具宏
